## 1. GitHub Pagesのセットアップ

### 1-1. GitHubアカウントの準備
- GitHubアカウントがない場合は作成: https://github.com/
- 既にアカウントがある場合はそのまま使用

### 1-2. リポジトリの作成

**手順:**
1. GitHubにログイン
2. 右上の「+」→「New repository」をクリック
3. リポジトリ名を入力（例: `meguro8bundan-website`）
4. Public（公開）を選択
5. 「Add a README file」にチェック（任意）
6. 「Create repository」をクリック

**リポジトリ命名のポイント:**
- 半角英数字とハイフンのみ使用
- 分かりやすい名前を推奨
- 例: `meguro8bundan`, `meguro-fire-brigade-8`

### 1-3. GitHub Pagesの有効化

**手順:**
1. 作成したリポジトリのページを開く
2. 「Settings」タブをクリック
3. 左サイドバーから「Pages」をクリック
4. 「Source」セクションで以下を選択:
   - Branch: `main`（または `master`）
   - Folder: `/ (root)`
5. 「Save」をクリック

**完了すると:**
- 数分後にサイトが公開される
- URLが表示される（例: `https://username.github.io/repository-name/`）

## 2. サイトファイルのアップロード

### 2-1. GitHub Desktopの使用（推奨）

**GitHub Desktopのインストール:**
1. https://desktop.github.com/ からダウンロード
2. インストール後、GitHubアカウントでサインイン

**リポジトリのクローン:**
1. GitHub Desktopを起動
2. 「File」→「Clone repository」
3. 作成したリポジトリを選択
4. ローカルの保存先を選択
5. 「Clone」をクリック

**ファイルのアップロード:**
1. クローンしたフォルダに、作成したサイトファイルをコピー
   ```
   /repository-name/
   ├── index.html
   ├── activity.html
   ├── interview.html
   ├── training.html
   ├── faq.html
   ├── form.html
   ├── css/
   ├── js/
   └── images/
   ```

2. GitHub Desktopに戻る
3. 左側に変更されたファイルが表示される
4. 左下の「Summary」に変更内容を記入（例: "Initial site upload"）
5. 「Commit to main」をクリック
6. 上部の「Push origin」をクリック

**完了:**
- 数分後にGitHub Pagesに反映される

### 2-2. Git コマンドラインの使用（上級者向け）

```bash
# リポジトリをクローン
git clone https://github.com/username/repository-name.git
cd repository-name

# ファイルをコピー後
git add .
git commit -m "Initial site upload"
git push origin main
```

### 2-3. GitHub Web UIでの直接アップロード（小規模な変更向け）

1. GitHubのリポジトリページを開く
2. 「Add file」→「Upload files」
3. ファイルをドラッグ&ドロップ
4. 「Commit changes」をクリック

**注意:** 大量のファイルがある場合は、GitHub DesktopまたはGitコマンドを推奨

### 5-1. 公開日の設定

**公開目標日:** 2026年1月21日（火）

**公開方法:**
- GitHub Pagesは自動的に公開されるため、特別な操作は不要
- リポジトリをPublicに設定していれば、既に公開状態

### 5-2. カスタムドメインの設定（オプション）

**独自ドメインを使用する場合:**

1. ドメインを取得（お名前.com、ムームードメイン等）
2. GitHub Pagesの設定画面で「Custom domain」に独自ドメインを入力
3. DNSレコードを設定:
   - Aレコードを追加:
     ```
     185.199.108.153
     185.199.109.153
     185.199.110.153
     185.199.111.153
     ```
   - または、CNAMEレコードを追加:
     ```
     username.github.io
     ```
4. 「Enforce HTTPS」にチェックを入れる

**反映時間:**
- DNS設定の反映には数時間〜24時間かかる場合がある

### 5-3. URLの確認と共有

**公開URL:**
- デフォルト: `https://username.github.io/repository-name/`
- カスタムドメイン使用時: `https://your-domain.com/`

## 🚨 トラブルシューティング

### サイトが表示されない

**原因と対処法:**
1. **GitHub Pagesが有効化されていない**
   - Settings → Pages → Sourceを確認

2. **ファイル名が間違っている**
   - トップページは必ず `index.html` にする

3. **プッシュが完了していない**
   - GitHub Desktopで「Push origin」を確認

4. **反映に時間がかかっている**
   - 最大10分程度待つ

### CSSが適用されない

**原因と対処法:**
1. **CSSファイルのパスが間違っている**
   - 絶対パスではなく相対パスを使用
   - 例: `href="css/style.css"`

2. **ファイル名の大文字小文字**
   - GitHubは大文字小文字を区別する
   - `Style.css` と `style.css` は別ファイル

3. **ブラウザのキャッシュ**
   - スーパーリロード（Ctrl+Shift+R）を試す

### フォームが送信されない

**原因と対処法:**
1. **フォーム送信サービスの設定ミス**
   - FormspreeやEmailJSの設定を確認

2. **JavaScriptのエラー**
   - ブラウザの開発者ツール（F12）→Consoleでエラー確認

3. **メールアドレスの誤入力**
   - 送信先メールアドレスを再確認

### 画像が表示されない

**原因と対処法:**
1. **画像ファイルのパスが間違っている**
   - 相対パスを確認
   - 例: `src="images/H-01.png"`

2. **ファイルがアップロードされていない**
   - GitHubのリポジトリでファイルを確認

3. **ファイル名の大文字小文字**
   - `Image.png` と `image.png` は別ファイル

4. **ファイル形式の問題**
   - JPG、PNG、GIFなどの対応形式を使用
